Understanding Data Loss Alerts

What is Data Loss?

Data loss occurs when your experiment has periods with zero visitors or conversions, indicating missing or corrupted data.

Why This Matters

Data Loss Compromises Your Experiment:

  • • Reduced statistical power - Less data means less reliable results
  • • Biased estimates - Missing data can skew your conversion rate calculations
  • • Invalid statistical tests - Assumptions about data continuity are violated
  • • Questionable validity - Results may not represent true user behavior

Common Causes

Technical Issues

  • • Server downtime or outages
  • • Tracking code failures
  • • Database connection problems
  • • CDN or hosting issues
  • • Analytics platform problems

Data Processing Issues

  • • Data pipeline failures
  • • ETL process errors
  • • Data corruption during transfer
  • • Incorrect data filtering
  • • Time zone conversion errors

Impact on Results

How data loss affects your experiment:

  • • Reduced sample size - Lower statistical power to detect effects
  • • Biased conversion rates - Missing data points can skew calculations
  • • Invalid confidence intervals - Statistical assumptions are violated
  • • Unreliable p-values - Test results may not be trustworthy

What To Do

1. Investigate the Root Cause

Check your analytics platform, server logs, and data pipelines to identify what caused the data loss.

2. Fix the Technical Issue

Resolve the underlying problem to prevent future data loss and ensure reliable data collection.

3. Consider Extending the Experiment

If the data loss is significant, consider running the experiment longer to compensate for lost data.

4. Document the Issue

Record what caused the data loss and implement monitoring to prevent similar issues in future experiments.

5. Assess Data Quality

If data loss is extensive, consider whether the remaining data is still reliable for decision-making.
