Overdue Experiments

An overdue experiment is one that has run longer than originally planned and collected more data than intended.

What Makes an Experiment Overdue?

  • The experiment has run past its planned end date
  • Both variants have reached the required sample size at a full-week boundary
  • Additional data has been collected beyond the planned horizon

Why This Matters

When an experiment runs longer than planned, it can introduce several issues:

  • Peeking Bias: Additional data collection can lead to p-hacking and false discoveries
  • Seasonal Effects: Extended runtime may capture different seasonal patterns
  • Sample Bias: Later data may come from different user segments
  • Statistical Power Inflation: More data than planned can inflate statistical significance

How We Handle Overdue Experiments

Our analysis automatically filters overdue experiments to ensure reliable results:

  1. Identify the Planned End: Find the original planned end date
  2. Check Sample Size: Determine if both variants reached required sample size at a full-week boundary
  3. Filter Data: Use only data up to the planned end date (or until sample size was met)
  4. Re-analyze: Perform statistical analysis on the filtered dataset

Best Practices

  • Stick to the Plan: End experiments at their planned end date
  • Pre-define Success Criteria: Set clear rules for when to stop
  • Avoid Peeking: Don't check results before the planned end
  • Document Extensions: If you must extend, document the reasons and adjust analysis accordingly

Remember: The goal is to make decisions based on the data you planned to collect, not the data you actually collected.
